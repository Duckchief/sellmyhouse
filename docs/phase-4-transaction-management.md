## Phase 4: Transaction Management

### 4.1 Offer Management
- Record offers, AI-assisted analysis (offer vs market median), agent review before sharing, counter-offer tracking, seller notification on new offers

### 4.2 Portal-Ready Listing Output
PropertyGuru, 99.co, and SRX do NOT offer public listing creation APIs. The platform generates portal-ready content for manual posting.

**`portalFormatter` service:** Takes approved listing → generates structured JSON for each portal (title, description, flat details, photos, CEA details) → agent views in dashboard with "Copy to Clipboard" per field → agent posts manually to each portal → marks as posted and pastes live URL back into system.

### 4.3 OTP Management (Physical Form Tracking)
The OTP is HDB's physical printed form with unique serial number. Platform TRACKS the process, does not generate the document.

**Status flow:** `prepared → sent_to_seller → signed_by_seller → returned → issued_to_buyer → exercised / expired`

- Agent updates status at each step → timestamp + audit log + notification to seller
- At "signed_by_seller" or "returned": agent uploads scanned copy (PDF/image) to `/uploads/otp/`
- At "issued_to_buyer": exercise deadline countdown starts (default 21 calendar days)
- Automated reminders: 14, 7, 3, 1 day(s) before deadline (WhatsApp + email + in-app)
- Agent must confirm OTP review (mandatory gate) before advancing to "issued_to_buyer"

### 4.4 Commission Invoice (Huttons Upload)
Invoice generated by Huttons' system, not this platform.

**Flow:** Transaction completes → agent uploads Huttons invoice PDF → enters invoice reference → sends to client via WhatsApp/email with PDF attachment → tracks payment status.

**Fixed amounts:** $1,499 + $134.91 GST = $1,633.91 (display values only, actual invoice from Huttons)

### 4.5 Completion & Post-Completion
- HDB application tracking (status-based, agent updates manually)
- Appointment date management
- Post-completion sequence: thank-you (day 1), testimonial request (day 7), buyer-side follow-up (day 14, marketing consent required)

### 4.7 Negotiation Recording
Negotiation happens outside the platform (WhatsApp, phone calls). The agent records the outcome in the system.

- Agent updates `Offer` status: pending → countered / accepted / rejected
- When countering: agent enters counter-offer amount, system creates a new `Offer` record linked to the original (via `parentOfferId` FK — add to schema)
- Chain of offers is visible in seller detail view: Offer $620K → Counter $650K → Counter $640K → Accepted $645K
- Agent can add notes to each offer/counter (e.g., "Buyer willing to accept $645K if completion within 8 weeks")
- When offer is accepted: system prompts agent to create Transaction record with the agreed price

**Add to Offer model:** `parentOfferId` (FK, nullable — links counter-offers to original), `counterAmount` (nullable — the counter-offer amount)
