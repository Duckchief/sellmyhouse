# Phase 1a: Full Prisma Schema — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand the V2 Prisma schema from bootstrap (SystemSetting + AuditLog) to the full application schema with all models, enums, relations, and indexes. Use cuid2 IDs, Decimal for money, snake_case DB mapping, and typed enums.

**Architecture:** Single `prisma/schema.prisma` file with all models. IDs are cuid2 strings generated in application code (no `@default()` on IDs). All table/column names use `@map`/`@@map` for snake_case in DB while keeping camelCase in TypeScript. Money fields use `Decimal` type. Enums are Prisma native enums mapped to snake_case. Relations preserve the V1 structure with cuid2 IDs replacing UUIDs.

**Tech Stack:** Prisma 6, PostgreSQL 16, TypeScript 5

**Spec:** `docs/superpowers/specs/2026-03-10-v2-rewrite-design.md`
**V1 reference:** `../sellmyhouse/prisma/schema.prisma` (original schema), `../sellmyhouse/docs/phase-0-shared-context.md` (schema spec)
