# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a light/dark mode toggle to all three authenticated portals (seller, agent, admin) with localStorage persistence and system preference detection.

**Architecture:** Tailwind `darkMode: 'class'` with CSS custom properties for panel/surface backgrounds and `dark:` utility overrides for text/borders. An inline script in `<head>` prevents flash of unstyled content on page load. A single toggle button in the shared `top-header.njk` drives the feature across all portals via the existing event-delegation system in `app.js`.

**Tech Stack:** Tailwind CSS v3 (class-based dark mode), CSS custom properties, vanilla JS, Nunjucks

**Spec:** `docs/superpowers/specs/2026-03-18-dark-mode-design.md`

### Task 1: Configure Tailwind dark mode and CSS variable tokens

**Files:**
- Modify: `tailwind.config.ts`
- Modify: `src/views/styles/input.css`

- [ ] **Step 1: Enable `darkMode: 'class'` and add `panel` color token in `tailwind.config.ts`**

Replace the entire file with:

```ts
import type { Config } from 'tailwindcss';

const config: Config = {
  content: ['./src/views/**/*.njk', './src/views/**/*.html'],
  darkMode: 'class',
  theme: {
    extend: {
      colors: {
        ink: '#1a1a2e',
        accent: {
          DEFAULT: '#c8553d',
          dark: '#a8432f',
        },
        bg: {
          DEFAULT: 'var(--color-surface)',
          alt: 'var(--color-surface-alt)',
        },
        panel: 'var(--color-panel)',
      },
    },
  },
  plugins: [],
};

export default config;
```

The `bg` color tokens now reference CSS vars (so `bg-bg` switches automatically in dark mode). The new `panel` token is used for card/header/dropdown/sidebar surfaces.

- [ ] **Step 2: Add CSS variable definitions to `src/views/styles/input.css`**

Insert after the `@tailwind utilities;` line and before `@layer utilities {`:

```css
@layer base {
  :root {
    --color-surface:     #fafaf7;
    --color-surface-alt: #f0efe9;
    --color-panel:       #ffffff;
  }
  .dark {
    --color-surface:     #111111;
    --color-surface-alt: #161616;
    --color-panel:       #1e1e1e;
  }
}
```

So the file reads:
```
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base { ... vars ... }
@layer utilities { ... animations ... }
@layer components { ... }
```

- [ ] **Step 3: Add `dark:` variants to the component classes in `input.css`**

Update `.input-field`, `.card`, and `.page-section-title` inside `@layer components`:

### Task 2: FOUC prevention and body dark class in base layout

**Files:**
- Modify: `src/views/layouts/base.njk`

- [ ] **Step 1: Add inline FOUC-prevention script and dark body class**

The script must run before the CSS link so that the `dark` class is on `<html>` before the browser computes styles. Add it as the very first tag inside `<head>`, and add `dark:text-gray-100` to `<body>`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <script nonce="{{ cspNonce }}">(function(){var s=localStorage.getItem('theme'),p=window.matchMedia('(prefers-color-scheme: dark)').matches;if(s==='dark'||(s===null&&p))document.documentElement.classList.add('dark');}());</script>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{% block title %}SellMyHomeNow.sg{% endblock %}</title>
  <link rel="stylesheet" href="/css/output.css">
  <script src="/js/htmx.min.js" nonce="{{ cspNonce }}"></script>
  <link rel="icon" href="/icons/favicon.svg" type="image/svg+xml">
  <link rel="manifest" href="/manifest.json">
  <meta name="theme-color" content="#1a1a2e">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  <link rel="apple-touch-icon" href="/icons/icon-192.svg">
  {% block head %}{% endblock %}
</head>
<body class="min-h-screen bg-bg text-gray-900 dark:text-gray-100">
  {% block body %}{% endblock %}
  {% include "partials/cookie-consent.njk" %}
  <script src="/js/app.js"></script>
</body>
</html>
```

- [ ] **Step 2: Add `toggle-dark-mode` handler and system preference listener to `app.js`**

Add the system preference listener after the `formLoadedAt` IIFE and before the `// ── Click event delegation` comment:

```js
  // ── Dark mode: system preference live listener ─────────────────
  (function () {
    window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', function (e) {
      if (!localStorage.getItem('theme')) {
        document.documentElement.classList.toggle('dark', e.matches);
      }
    });
  })();
```

Add the toggle handler inside the click delegation block (after the `toggle-referral-message` block, before the closing `});`):

```js
    // Toggle dark mode
    if (action === 'toggle-dark-mode') {
      var isDark = document.documentElement.classList.toggle('dark');
      localStorage.setItem('theme', isDark ? 'dark' : 'light');
    }
```
